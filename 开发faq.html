JavaScript heap out of memory

一般可通过设置 --max-old-space-size 解决，例如修改为 6G，有两种方式：
  1. 全局方式，修改 ~/.zshrc 文件：
      2. export NODE_OPTIONS=--max-old-space-size=6144
 
  1. 本地方式，使用于某些具体的项目：
      2. NODE_OPTIONS=--max-old-space-size=6144 yarn start

// 或者 在package.json 中配置
{
  "scripts": {
     "start": "NODE_OPTIONS=--max-old-space-size=6144 webpack"
  }
}
       

      不同项目使用不同的Node版本

      因为习惯是使用VSCode进行开发，如果需要使用一些命令行也是直接使用VSCode的Terminal，一个简单思路，初始化Terminal时，自动切换指定的Node的版本：
      // .zshrc 文件添加

use-node14() {
 local project_path
 project_path="old-project-fe"

 if [[ $PWD =~ $project_path ]]; then
   nvm use 14
 fi
}
use-node14
      通过目录路径匹配关键字的方式实现Node版本的切换，需要注意只有在初始化Terminal时才会执行。
       
      另一种方式是通过 .nvmrc 指定目录需要使用的Node版本，具体可以参考 https://github.com/nvm-sh/nvm#calling-nvm-use-automatically-in-a-directory-with-a-nvmrc-file
